# Reference

The reference section is the structured surface of Beluga's documentation. It is the place to look up an exact function signature, see the full list of providers in a category, or trace which architectural document explains a specific design decision.

## Sections

| Section | What's in it |
|---|---|
| [Providers](/docs/reference/providers/) | All 110 providers across 19 categories — LLM, embedding, vector store, voice, guard, workflow, observability, and more. |
| [API](/docs/reference/api/) | Generated API documentation per package — foundation, LLM and agents, memory and RAG, voice, infrastructure, protocol and server. |
| [Architecture](/docs/reference/architecture/) | The 7-layer model, the package dependency map, the four extensibility rings, and the provider integration model. |
| [Config Schema](/docs/reference/config-schema/) | Every `WithX()` option across the framework, with defaults and constraints. |

## When to use Reference

- You know the function name and want the signature.
- You need the complete list of providers in a category.
- You want to verify a claim made in a guide or concept page against the source.

## When to use something else

- You want to learn the framework — start with [Concepts](/docs/concepts/).
- You want to build a feature — go to [Guides](/docs/guides/).
- You want a copy-paste snippet — try [Recipes](/docs/recipes/).
