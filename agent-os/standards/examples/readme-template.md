# Template-Driven README Structure

All example READMEs follow a strict template for consistency.

## Required Sections

```markdown
## Description
This example shows you how to {what}. You'll learn:
- {Skill 1}
- {Skill 2}

## Prerequisites
| Requirement | Version | Notes |
|-------------|---------|-------|
| Go | 1.24+ | [Install Go](https://go.dev/doc/install) |

## Usage
### Running the Example
1. Clone the repository
2. Install dependencies
3. Run the example

### Expected Output
{Console output sample}

## Code Structure
{Directory tree and file descriptions}

## Testing
{How to run tests}

## Troubleshooting
<details>
<summary>Error: "common error"</summary>
**Cause:** {Why}
**Solution:** {Fix}
</details>

## Related Examples
- **[Next Example](../path/)** - {Description}
```

## Opening Convention
- Start description with "This example shows you how to..."
- Use active voice throughout
- Include "You'll learn:" bullet list

## Table Format
- Use tables for prerequisites, command-line options, code structure
- Consistent columns: Requirement/Version/Notes or Flag/Description/Default
